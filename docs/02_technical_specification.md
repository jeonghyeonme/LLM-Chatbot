# 🛠️ 기술 사양 및 구현 전략 (교안 기반)

본 프로젝트는 제공된 교안의 핵심 기술들을 활용하여 구현됩니다.

## 1. LLM 및 API 활용 (03, 04주차 교안)
- **Model**: OpenAI `gpt-4o` 또는 `gpt-3.5-turbo`
- **구현**: 
  - 각 페르소나별로 독립적인 `System Message`를 부여하여 대화 일관성 유지.
  - 두 페르소나의 답변을 병렬(Parallel)로 호출하여 실시간성 확보.

## 2. 상호작용 제어: Function Calling (07주차 교안)
- **기능**: 단순히 답변을 출력하는 것이 아니라, 필요 시 `interact_with_partner` 함수를 호출.
- **로직**:
  - 페르소나 A가 답변 후, 대화의 맥락상 B의 의견이 필요하다고 판단되면 함수 호출.
  - 함수 인자로 현재까지의 대화 내용과 토론하고 싶은 주제를 전달.
  - 오케스트레이터(Orchestrator)가 이를 중재하여 두 에이전트 간의 대화를 생성.

## 3. 음성 상담 인터페이스 (05주차 교안)
- **STT (Speech-to-Text)**: OpenAI `Whisper API`를 사용하여 사용자의 음성 고민을 텍스트로 변환.
- **가치**: 텍스트 입력보다 감정 전달이 풍부한 음성 기반 상담으로 사용자 경험 강화.

## 4. 데이터 및 정보 활용 (08주차 교안)
- **외부 데이터 연동**: 필요 시 Yahoo Finance 등을 참고하여 취업/경제 고민에 대한 시장 데이터 기반의 조언 제공.

## 5. 시스템 구조 (Architecture)
1. **User Input**: 사용자가 고민 입력 (텍스트/음성).
2. **Orchestrator**: 입력을 분석하고 두 페르소나에게 전달.
3. **Dual Generation**: 엘리아와 루나가 각각의 관점에서 답변 생성.
4. **Interaction Check**: 두 페르소나가 서로 대화할 필요가 있는지 판단.
5. **Final Output**: 페르소나 A의 답변, 페르소나 B의 답변, 그리고 그들의 짧은 상호작용을 화면에 출력.
